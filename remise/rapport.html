Tâche 1: Implémentation

Notes sur notre implémentation: hypothèses, décisions, etc:

Le code source est placé directement dans le jar sous “main/java” et “test/java”. Lien vers le jar
Les sources sont aussi dans Implementation/src, avec les tests. Nos dpendences sont dans Implementation/Dependencies

  1. Pour les paies, on a supposé que si un professionnel est payé par le nombre de clients, c’est qu’il est en fait payé pour les clients qui se sont présentés à l’activité. On ne regarde donc pas les inscriptions pour la paie: on regarde les Attendance pour voir qui était présent la journée même
  2. On utilise la réflection pour améliorer l’agilité du projet: lorsque les besoins changent souvent, il est beaucoup plus simple d’utiliser la réflection pour gérer le lien entre l’interface usager et le contrôleur (le Prototype) que d’appeler explicitement toutes les méthodes nécessaires
  3. Lorsqu’on supprime une instance d’Activity, Profesionnal, ou Client, on ne cherche pas dans tous les endroits qui y ont référence: on ne l’enlève que dans le Datastore. Et ça fait du sens: si on enlève une activité, on veut quand même que les clients aient à payer s’ils y ont participé lors de la semaine, etc.
  4. Les code QR ne sont pas imprimés à l’écran: ils sont placés sur disque dans un fichier qui s’appelle “QRCODE.jpg”. Dans la vraie vie, ils seraient envoyés par internet aux applications des gens concernés.
  5. Dans la vraie vie, le Datastore serait séparé en plusieurs classes qui gèrent les Clients, les Activités, et les Professionnels. Mais ici, comme on n’a pas de base de donnée SQL, on a tout mit dans une seule classe.
  6. Lorsqu’on demande un UUID à l’usager, il peut entrer le chiffre correspondant à l’ID qu’il veut vraiment entrer. Donc, par exemple, “000000001” peut aussi être entré comme “1”. (On enlève les “leading zeros”).

Guide d’utilisation

Interface principale

La première étape pour chaque élément de ce guide réfère à sélectionner une lettre dans l’interface principal dont la capture d’écran est ci-dessous.

En tout temps (lorsque vous êtes dans l’interface principal), vous pouvez sortir du programme en entrant L puis en appuyant sur ENTRÉE.

Fonctionnalités d’ajout/modification/suppression

Inscrire un Membre, professionnel, activité:

  1. Après avoir lancé le programme vous écrivez dans le terminal la lettre A, B pi E correspondant a Membre , Professionnel ou Activité,  puis appuyez sur ENTRÉE
  2. On vous demande maintenant l’action à effectuer; appuyez sur la touche 1 puis ENTER
  3. Il vous sera ensuite demandé de rentrer les informations une à une. une fois l’information entrée, appuyez sur entrée.
  4. Un message de confirmation va s’afficher et vous pourrez continuer à utiliser le logiciel en appuyant sur ENTER

Modifier un Membre, Professionnel, Activité

  1. Sélectionnez A, B ou E dépendamment si vous voulez Membre, Professionnel ou Activité.
  2. Sélectionnez l’option 2
  3. Chaque champ vous sera demandé , entrez “NC” si vous ne voulez pas modifier le champ , et entre “ la nouvelle valeure “ si vous voulez modifier le champ
  4. Un message de confirmation s’affiche et l’entité ou activité est modifiée

Supprimer un Membre, Professionnel, Activité

  1. Sélectionnez A, B ou E dépendamment si vous voulez Membre, Professionnel ou Activité.
  2. Sélectionnez l’option 3
  3. On vas vous afficher une liste des éléments avec leur id.
  4. Choisissez le id de l'élément que vous voulez supprimer et entrez-le puis appuyez sur ENTRÉE.
  5. Le Membre, Professionnel ou l’activité est maintenant supprimé/e.

Fonctionnalités quotidiennes du GYM

Inscrire un membre à une activité

  1. Pour inscrire un membre à une activité, sélectionner l’option C puis appuyez sur ENTRÉE
  2. On vous demandera le ID du membre.
  3. La liste des activités va s’afficher
  4. Entrez le ID de l’activité.
  5. On vous demandera à quelle date il veut y participer.
  6. Après avoir entré toutes les informations demandées, le logiciel vous affichera un message signifiant la réussite ou l’échec avec les explications.

Faire accéder un membre au gym

  1. Pour faire accéder un membre au gym, sélectionnez l’option D puis appuyez sur ENTRÉE.
  2. Entrez le ID du client après lui avoir demandé.
  3. Le dym va vérifier si le client est bien inscrit et en règles et affichera un message vous indiquant si il peut maintenant accéder ou pas avec des explications.

Consulter les activités libres

  1. Sélectionnez F
  2. Les activités vont s’afficher à l’écran

Consulter les inscriptions

  1. Sélectionnez G
  2. Entrez le numéro du professionnel qui veut consulter ses inscriptions
  3. Les inscriptions seront affichées à l’écran

Confirmer la présence d’un membre à une activité

  1. Sélectionnez H
  2. La liste des activités va s’afficher à l’écran
  3. Entrez l’ID du membre qui veut s’inscrire ainsi que l’ID de l’activité et un commentaire

Fonctionnalités des rapports du GYM

Imprimer le rapport des entités (client ou professionnel)

  1. Sélectionnez K

  1. Sélectionnez 1 pour le client, 2 pour le professionnel.
  2. Entrez l’ID de l’entité
  3. Le rapport sera imprimé sous le nom de fichier “<Nom de l’entité><date d’aujourd’hui>”.

Imprimer le rapport du manager

  1. Sélectionnez I
  2. Le rapport comptable sera imprimé sous le nom de fichier “report.tsv”

Imprimer le rapport du vendredi (celui du manager et les TEF)

  1. Sélectionnez J
  2. Le rapport comptable sera imprimé sous le nom de fichier “report.tsv”. Les TEF seront imprimés sous le nom de fichier “<UUID du professionnel>.tef”.

 Cas d'utilisation : Inscrire membre au #GYM.

    But : Le client a l’intention de s’inscrire au #GYM

    Préconditions : le client n’est pas déjà inscrit. Le client est en âge de s’inscrire à un

                                gym.

    Acteurs : Le client (principal), l’agent (secondaire).

    Scénario principal :

    1. Le client se présente à la réception du centre.

    2. Le client exprime à l’agent son intention de s’inscrire au #GYM.

    3. L’agent l’inscrit au #GYM (voir CU “inscrire une entité”)

      3.1 Un champ “isSuspended” est ajouté à l’entité pour indiquer si le client est valide

            (initialement valide)

    4. Le client est maintenant inscrit au gym et considéré comme un membre.

    Scénarios alternatifs : 

    4a.1. Le système notifie l’agent que l’utilisateur est déjà dans le système

    4a.2. L’agent rappel au membre son numéro unique

    4a.3. Comme le client était déjà inscrit au gym il n’est pas réinscrit.

    Postconditions : Le client est inscrit au gym

Cas d'utilisation : Inscrire entité au #GYM.

    But : L’entité veut s’inscrire au #GYM

    Préconditions : L’entité respecte les conditions d’inscription.

    Acteurs : L’entité (client ou membre) (principal), l’agent (secondaire).

    Scénario principal : 

  1. L’Entité se présente au DYM pour s’inscrire.
  2. L’agent sélectionne l’option ‘’inscrire un nouveau membre’’ sur le système.
  3. L’agent demande les informations personnelles de l’entité.
  4. L’entité fournit à l’agent ses informations.
  5. L’agent enregistre les informations personnelles du client dans le système central via son ordinateur. Comme toutes les autres actions de création incluent l’étampe de temps, on assume que cette action l’inclue aussi. On inclut aussi un commentaire: en effet, il est important qu’un gym sache les allergies de ses clients, ou leurs handicaps (comme l'épilepsie, etc).
  6. Ceci crée la nouvelle entité et lui assigne un numéro unique.
  7. L’inscription est réussie et le système affiche ‘’inscription réussie’’ suivi du numéro unique.

    Scénarios alternatifs : 

    6a.1. Le système notifie l’agent que l’entité est déjà dans le système

    6a.2. L’agent rappelle à l’entité son numéro unique

    6b.1.Le client n’a pas toutes les informations nécessaires a son inscription.

    6b.2.Le client n’est pas inscrit au #GYM

    Postconditions : L’entité est inscrite au gym

    But : Le client veut consulter les activités/séances disponibles

    Préconditions : Le client a accès au gym

    Scénario principal :

    1. Le membre se présente à la réception.

    2. L’agent s’authentifie et affiche les activités libres. (voir CU “Consulter et filtrer le Répertoire des

    Scénarios alternatifs : 

    3.a Le client change d’idée et ne consulte pas la liste

    Postconditions : aucune

    NOTE: On aurait pu penser que ce CU était un simple include dans “S’inscrire à une activité”. Cependant, on remarque que dans “S’inscrire à une activité”, on mentionne qu’il faut pouvoir “confirmer son choix”. Il serait donc logique qu’en fait consulter la liste d’activités est une action séparée de l’inscription, et que ce n’est qu’après qu’on s’inscrit. De plus, ceci ferait beaucoup de sens: lorsqu’on va à un gym sportif dans la vraie vie, par exemple sur le site web du Cepsum, on peut consulter les activités possibles sans avoir même l’intention de s’y inscrire. C’est aussi pour cela que dans “S’inscrire à une activité”, avoir consulté les activités est une précondition.
